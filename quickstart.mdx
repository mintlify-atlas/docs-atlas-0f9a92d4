# Quickstart guide

This guide will help you make your first stroke risk prediction using the API. You can run the API locally or use the live deployment.

## Option 1: Use the live API

The fastest way to get started is to use the publicly deployed API.

<Steps>
  <Step title="Send a prediction request">
    Use curl to send a POST request to the live API endpoint:

    ```bash
    curl -X 'POST' \
      'https://api.stroke-predictor.acel.dev/predict' \
      -H 'accept: application/json' \
      -H 'Content-Type: application/json' \
      -d '{
            "age": 67,
            "gender": "Female",
            "hypertension": 0,
            "heart_disease": 1,
            "ever_married": "Yes",
            "work_type": "Private",
            "residence_type": "Urban",
            "avg_glucose_level": 228.69,
            "bmi": 36.6,
            "smoking_status": "formerly smoked"
          }'
    ```
  </Step>

  <Step title="View the response">
    The API will return a JSON response with the stroke probability and prediction:

    ```json
    {
      "stroke_probability": 0.234,
      "stroke_prediction": true
    }
    ```

    - `stroke_probability` - Float between 0 and 1 representing the predicted stroke risk
    - `stroke_prediction` - Boolean indicating whether the risk exceeds the 0.09 threshold
  </Step>

  <Step title="Try the interactive docs">
    Visit the Swagger UI at [api.stroke-predictor.acel.dev/docs](https://api.stroke-predictor.acel.dev/docs) to:
    
    - See all available endpoints
    - View dropdown menus for valid categorical values
    - Test requests directly in your browser
  </Step>
</Steps>

## Option 2: Run locally with Docker

<Note>
This is the recommended approach for local development. You'll need Docker installed on your machine.
</Note>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone git@github.com:Acel-01/stroke-risk-predictor.git
    cd stroke-risk-predictor
    ```
  </Step>

  <Step title="Build the Docker image">
    ```bash
    docker build -t stroke-predictor-api .
    ```

    The Dockerfile uses Python 3.10-slim as the base image and installs all dependencies using `uv` for fast, reproducible builds.
  </Step>

  <Step title="Run the container">
    ```bash
    docker run --rm -p 8000:8000 --name stroke_predictor stroke-predictor-api
    ```

    The API will be available at `http://127.0.0.1:8000`
  </Step>

  <Step title="Make a prediction">
    ```bash
    curl -X 'POST' \
      'http://127.0.0.1:8000/predict' \
      -H 'accept: application/json' \
      -H 'Content-Type: application/json' \
      -d '{
            "age": 67,
            "gender": "Female",
            "hypertension": 0,
            "heart_disease": 1,
            "ever_married": "Yes",
            "work_type": "Private",
            "residence_type": "Urban",
            "avg_glucose_level": 228.69,
            "bmi": 36.6,
            "smoking_status": "formerly smoked"
          }'
    ```
  </Step>
</Steps>

## Option 3: Run locally with uv

If you prefer to run without Docker:

<Steps>
  <Step title="Clone and navigate to the project">
    ```bash
    git clone git@github.com:Acel-01/stroke-risk-predictor.git
    cd stroke-risk-predictor
    ```
  </Step>

  <Step title="Set up the environment">
    ```bash
    uv venv
    source .venv/bin/activate
    uv pip sync
    ```

    This creates a virtual environment and installs all dependencies from the lockfile.
  </Step>

  <Step title="Start the API server">
    ```bash
    uvicorn serve:app --host 0.0.0.0 --port 8000
    ```

    The server will start at `http://127.0.0.1:8000`
  </Step>

  <Step title="Access the interactive docs">
    Navigate to `http://127.0.0.1:8000/docs` to see the Swagger UI and test the API.
  </Step>
</Steps>

## Understanding the response

The API returns two key values:

```python
{
  "stroke_probability": 0.234,  # Raw model output (0.0 to 1.0)
  "stroke_prediction": true     # Binary decision at 0.09 threshold
}
```

<Info>
The 0.09 threshold was optimized to maximize F1-Score on the validation set, balancing precision and recall for the imbalanced dataset.
</Info>

### Example predictions

**High-risk patient:**
```json
{
  "age": 67,
  "gender": "Female",
  "hypertension": 0,
  "heart_disease": 1,
  "bmi": 36.6,
  "smoking_status": "formerly smoked"
}
```
Typically returns: `stroke_probability: 0.23`, `stroke_prediction: true`

**Low-risk patient:**
```json
{
  "age": 23,
  "gender": "Male",
  "hypertension": 0,
  "heart_disease": 0,
  "bmi": 30.0,
  "smoking_status": "never smoked"
}
```
Typically returns: `stroke_probability: 0.02`, `stroke_prediction: false`
