This guide walks you through setting up the Stroke Risk Predictor API for local development using `uv`, a fast Python package manager.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3.10 or higher**: Check with `python --version`
- **uv package manager**: Install from [uv documentation](https://github.com/astral-sh/uv)
- **Git**: For cloning the repository

<Info>
If you prefer containerized deployment without local Python setup, see the [Docker deployment guide](/development/docker).
</Info>

## Installation steps

<Steps>
  <Step title="Clone the repository">
    Clone the project from GitHub:
    
    ```bash
    git clone git@github.com:Acel-01/stroke-risk-predictor.git
    cd stroke-risk-predictor
    ```
    
    You can also use HTTPS:
    ```bash
    git clone https://github.com/Acel-01/stroke-risk-predictor.git
    cd stroke-risk-predictor
    ```
  </Step>

  <Step title="Create a virtual environment">
    Create a new virtual environment using `uv`:
    
    ```bash
    uv venv
    ```
    
    This creates a `.venv` directory in your project root containing an isolated Python environment.
  </Step>

  <Step title="Activate the virtual environment">
    Activate the environment:
    
    ```bash
    source .venv/bin/activate
    ```
    
    <Note>
    On Windows, use `.venv\\Scripts\\activate` instead.
    </Note>
    
    Your terminal prompt should now show `(.venv)` indicating the environment is active.
  </Step>

  <Step title="Install dependencies">
    Sync all project dependencies from `pyproject.toml`:
    
    ```bash
    uv pip sync
    ```
    
    This installs all required packages including:
    - **FastAPI**: Web framework for building the API
    - **Uvicorn**: ASGI server for running FastAPI
    - **Pydantic**: Data validation using Python type annotations
    - **scikit-learn**: Machine learning library for the logistic regression model
    - **NumPy & Pandas**: Data processing
    - **XGBoost**: Alternative model (not used in production)
  </Step>
</Steps>

## Required files

The API requires these files in the project root:

<CodeGroup>

```plaintext Python files
serve.py         # FastAPI application and endpoints
predict.py       # Prediction logic and model loading
schemas.py       # Pydantic models for request validation
```

```plaintext Model files
model.bin        # Trained logistic regression model (pickled)
dv.bin           # DictVectorizer for feature transformation (pickled)
```

</CodeGroup>

<Warning>
The model files (`model.bin`, `dv.bin`) are required for the API to function. These are generated by the training process in `train.py` and must be present in the project root.
</Warning>

## Run the API server

Start the development server with `uvicorn`:

```bash
uvicorn serve:app --host 0.0.0.0 --port 8000
```

<Info>
**Understanding the command:**
- `serve:app`: Load the `app` object from `serve.py`
- `--host 0.0.0.0`: Accept connections from any network interface
- `--port 8000`: Run on port 8000
</Info>

You should see output like:

```plaintext
INFO:     Started server process [12345]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

### Enable hot reload

For development, add the `--reload` flag to automatically restart the server when code changes:

```bash
uvicorn serve:app --host 0.0.0.0 --port 8000 --reload
```

## Access the API

Once the server is running, you can interact with the API in several ways:

### Swagger UI (Interactive docs)

Navigate to **http://127.0.0.1:8000/docs** in your browser.

The Swagger UI provides:
- Interactive API documentation
- Schema definitions for all endpoints
- Dropdown menus for categorical fields (gender, work_type, etc.)
- Try-it-out functionality to test requests directly

### Health check endpoint

Test the health check endpoint:

```bash
curl http://127.0.0.1:8000/
```

Expected response:
```json
{
  "status": "ok",
  "message": "Stroke prediction service is running."
}
```

### Prediction endpoint

Send a prediction request using `curl`:

```bash
curl -X 'POST' \
  'http://127.0.0.1:8000/predict' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
        "age": 67,
        "gender": "Female",
        "hypertension": 0,
        "heart_disease": 1,
        "ever_married": "Yes",
        "work_type": "Private",
        "residence_type": "Urban",
        "avg_glucose_level": 228.69,
        "bmi": 36.6,
        "smoking_status": "formerly smoked"
      }'
```

<CodeGroup>

```json Example response
{
  "stroke_probability": 0.4523,
  "stroke_prediction": true
}
```

```json Response schema
{
  "stroke_probability": "float (0-1)",
  "stroke_prediction": "boolean (true if probability ≥ 0.09)"
}
```

</CodeGroup>

## Understanding the request schema

The `/predict` endpoint expects a JSON object with these fields:

| Field | Type | Required | Example | Description |
|-------|------|----------|---------|-------------|
| `age` | float | Yes | `67.0` | Patient age |
| `gender` | string | Yes | `"Female"` | One of: `Male`, `Female`, `Other` |
| `hypertension` | integer | Yes | `0` | `0` = No, `1` = Yes |
| `heart_disease` | integer | Yes | `1` | `0` = No, `1` = Yes |
| `ever_married` | string | Yes | `"Yes"` | One of: `Yes`, `No` |
| `work_type` | string | Yes | `"Private"` | One of: `Private`, `Self-employed`, `Govt_job`, `children`, `Never_worked` |
| `residence_type` | string | Yes | `"Urban"` | One of: `Urban`, `Rural` |
| `avg_glucose_level` | float | No | `228.69` | Average glucose level (defaults to 106.45 if omitted) |
| `bmi` | float | Yes | `36.6` | Body mass index |
| `smoking_status` | string | Yes | `"formerly smoked"` | One of: `formerly smoked`, `never smoked`, `smokes`, `Unknown` |

<Note>
The `avg_glucose_level` field is optional. If not provided, the API uses the dataset mean value of 106.45.
</Note>

## How predictions work

The prediction process in `predict.py` follows these steps:

1. **Load models**: The pickled `model.bin` (logistic regression) and `dv.bin` (DictVectorizer) are loaded at import time
2. **Handle missing values**: If `avg_glucose_level` is null, it defaults to 106.45 (dataset mean)
3. **Transform features**: The DictVectorizer converts the input dictionary to a feature matrix
4. **Predict probability**: The model outputs a probability between 0 and 1
5. **Apply threshold**: A decision is made using the optimal threshold of 0.09 (determined during training)
6. **Return results**: Both the probability and boolean decision are returned

## Troubleshooting

### Port already in use

If port 8000 is already in use:

```bash
# Use a different port
uvicorn serve:app --host 0.0.0.0 --port 8001
```

### Model files not found

If you see `FileNotFoundError: [Errno 2] No such file or directory: 'model.bin'`:

1. Ensure you're in the project root directory
2. Run the training script to generate model files:
   ```bash
   python train.py
   ```

### Import errors

If you encounter import errors, ensure:
1. The virtual environment is activated (`source .venv/bin/activate`)
2. Dependencies are installed (`uv pip sync`)
3. You're using Python 3.10 or higher
